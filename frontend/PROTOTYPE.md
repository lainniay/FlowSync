# FlowSync 页面地图与低保真原型

本文档只确定页面结构、导航、权限、交互流程和反馈状态，不定义高保真视觉。

实现依据：

1. 根目录 `AGENTS.md`
2. `README.md`
3. `../docs/api.md`
4. `../docs/relationship.md`

## 1. 页面与权限原则

- 未登录用户只能访问登录页。
- `ADMIN` 是系统管理账号，可以查看全部项目数据并执行契约允许的管理操作。
- `ADMIN` 不能成为项目 owner、成员、被邀请人或任务负责人，不能创建项目内容。
- `USER` 的项目权限由 owner、member、assignee、creator 等资源关系决定。
- `systemRole=USER` 不等于项目 owner。
- 已归档项目只读，只允许恢复或永久删除。
- AI 输出只保留在页面临时状态，人工确认后才能通过普通接口写入业务记录。
- 页面不能通过隐藏按钮代替后端权限校验；后端仍是最终权限边界。

## 2. 根布局

登录后的所有业务页面共用 `AppLayout`。

```text
┌──────────────────────────────────────────────────────────────┐
│ FlowSync                         用户名 / 角色 / 退出         │ 顶栏
├──────────────────┬───────────────────────────────────────────┤
│ 工作台           │ 页面标题                    页面级主操作  │
│ 项目             │ 面包屑                                    │
│ 任务             ├───────────────────────────────────────────┤
│ 总结             │ 筛选区                                    │
│ 邀请或用户管理   ├───────────────────────────────────────────┤
│                  │                                           │
│ 个人中心         │ 内容区：表格、表单、详情或反馈状态         │
│                  │                                           │
└──────────────────┴───────────────────────────────────────────┘
```

布局边界：

- `AppLayout` 只负责顶栏、侧栏、用户入口、退出入口和业务内容容器。
- 业务页面通过子路由渲染到布局中的 `RouterView`。
- 业务页面不能重复实现顶栏、侧栏或全局退出逻辑。
- 页面标题、筛选区、表格区和页面操作属于具体业务页面。
- 项目级权限通过项目响应和当前用户关系判断，不能写死在根布局中。

## 3. 路由地图

建议使用以下路由结构：

| 路径 | 页面 | 访问条件 | 主要用途 |
| --- | --- | --- | --- |
| `/login` | LoginView | 公开 | 登录；已登录用户重定向到 `/overview` |
| `/` | AppLayout | 已登录 | 根布局，默认重定向到 `/overview` |
| `/overview` | OverviewView | 已登录 | 工作台统计和最近活动 |
| `/admin/users` | UserListView | ADMIN | 用户查询、创建、编辑、停用和重置密码 |
| `/projects` | ProjectListView | 已登录 | 项目搜索、筛选、分页和进入详情 |
| `/projects/:projectId` | ProjectDetailView | 成员或 ADMIN | 项目详情、任务、成员、邀请和总结入口 |
| `/projects/:projectId/ai-plan` | AiTaskPlanView | USER 且为 owner | 生成、人工编辑和导入临时任务计划 |
| `/tasks` | TaskListView | 已登录 | 跨项目任务查询和筛选 |
| `/tasks/:taskId` | TaskDetailView | 成员或 ADMIN | 任务详情、日志和 AI 建议入口 |
| `/summaries` | SummaryListView | 已登录 | 总结搜索、筛选和分页 |
| `/summaries/:summaryId` | SummaryDetailView | 成员或 ADMIN | 总结详情和授权操作 |
| `/invitations` | InvitationListView | USER | 查看并处理自己收到的邀请 |
| `/profile` | ProfileView | 已登录 | 修改个人资料和密码 |
| `/403` | ForbiddenView | 已登录 | 已登录但无页面权限 |
| `/:pathMatch(.*)*` | NotFoundView | 已登录 | 未匹配路由 |

路由只处理登录状态和系统角色。owner、member、assignee、creator 等资源权限在页面取得资源后判断。

## 5. 页面定义

### 5.1 登录页

```text
┌──────────────────────────────────┐
│ FlowSync                         │
│ 小组任务协同管理系统             │
│                                  │
│ 用户名 [______________________]  │
│ 密码   [______________________]  │
│                                  │
│ [              登录           ]  │
│                                  │
│ 登录错误或服务错误               │
└──────────────────────────────────┘
```

流程：

1. 打开 `/login`。
2. 输入用户名和密码。
3. 前端表单校验通过后请求 CSRF Token。
4. 调用登录接口。
5. 登录成功后进入原目标地址或 `/overview`。
6. 已登录用户访问 `/login` 时跳转到 `/overview`。

状态：

- 提交中：登录按钮 loading，禁止重复提交。
- 表单错误：显示在对应字段下。
- 凭据错误：显示服务端 Problem Details。
- 网络错误：显示通用错误，不误报为用户名或密码错误。

### 5.2 工作台

内容：

- 项目数、任务数、已完成任务数和逾期任务数
- 总结数和去重成员数
- 任务状态分布
- 最近活动
- 可选项目筛选

ADMIN 统计全部可见数据；USER 只统计自己可见的项目数据。工作台只读，不承担项目、任务或
总结编辑。

### 5.3 用户管理

仅 ADMIN 访问。

筛选区：用户名或显示名搜索、系统角色、启用状态、查询、重置和创建用户。

表格列：用户名、显示名称、系统角色、手机、邮箱、启用状态、创建时间和操作。

操作：

- 创建用户：对话框
- 编辑用户：完整 PUT 表单
- 启用或停用：确认对话框
- 重置密码：独立对话框
- 查看详情：抽屉或详情区

失败时展示 `LAST_ADMIN_REQUIRED`、`USER_OWNS_PROJECT`、`USER_HAS_ACTIVE_TASKS`、
`USER_HAS_PROJECT_MEMBERSHIP` 等具体错误。成功后重新查询当前页；若当前页因筛选条件变空，
应回到有效页码。

### 5.4 项目列表

这是后续业务模块复制的标准纵向示例。

```text
项目
┌────────────────────────────────────────────────────────────┐
│ 搜索 [________] 状态 [全部] Owner [全部] 归档 [否]         │
│ [查询] [重置]                                  [创建项目]  │
└────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────┐
│ 名称 │ Owner │ 状态 │ 优先级 │ 日期 │ 成员 │ 任务 │ 操作 │
├────────────────────────────────────────────────────────────┤
│ ...                                                        │
└────────────────────────────────────────────────────────────┘

共 N 条                         [上一页] 1 2 3 [下一页]
```

ADMIN 和 USER 都可访问：

- USER 只能看到自己参与的项目。
- ADMIN 可以看到全部项目。
- USER 创建项目时 `ownerId` 发送 `null`，当前用户自动成为 owner。
- ADMIN 创建项目时必须选择有效 USER 作为 owner。
- 行点击或“查看”进入 `/projects/:projectId`。

列表状态：

- 首次加载：表格 loading。
- 查询成功且有数据：表格和分页。
- 查询成功但为空：显示空状态；有筛选时提供“清除筛选”。
- 查询失败：显示错误说明和“重新加载”。
- 刷新：保留当前筛选和分页重新请求。

### 5.5 项目详情

```text
项目名称                         [编辑] [归档/恢复] [更多]
Owner / 状态 / 优先级 / 日期 / 是否归档

[概览] [任务] [成员] [邀请] [总结] [AI 计划]

当前标签页内容
```

标签页：

- 概览：描述、Owner、日期、成员数量和任务统计。
- 任务：当前项目任务列表，进入任务详情。
- 成员：成员列表。
- 邀请：owner 或 ADMIN 可查看。
- 总结：项目相关总结列表。
- AI 计划：仅当前 USER owner 显示。

权限差异：

- owner 或 ADMIN：编辑项目、转移 owner、归档、恢复和永久删除已归档项目。
- ADMIN：直接批量添加 USER 成员。
- owner：通过邀请批量邀请 USER，不能直接添加成员。
- owner 或 ADMIN：移除符合条件的成员。
- ADMIN：不能创建邀请、任务、日志、总结或 AI 内容。
- 普通成员：只查看项目和成员，可执行契约允许的项目内容操作。
- 已归档项目：所有内容写入口禁用，只保留恢复和永久删除。

对话框：

- 创建或编辑项目
- 转移 owner
- 归档、恢复和永久删除确认
- ADMIN 添加成员
- owner 创建邀请
- 移除成员或取消邀请确认

每次成功写入后重新查询项目详情及受影响的标签页数据。

### 5.6 AI 任务计划

仅 `systemRole=USER` 且为当前项目 owner 可进入。

```text
项目目标
[________________________________________________________]

补充说明
[________________________________________________________]

最大条目 [10]   目标结束日期 [________]

[生成初步计划]

临时计划：
┌──────────────────────────────────────────────────────────┐
│ 顺序 │ 标题 │ 父任务 │ 负责人 │ 优先级 │ 截止日期 │ 操作 │
└──────────────────────────────────────────────────────────┘

[新增临时条目] [重新生成] [确认并导入]
```

边界：

- 生成结果只保存在当前页面状态。
- 不创建 `planId`，不写 Pinia 持久状态，不使用 localStorage。
- owner 必须能编辑、增加、删除和调整条目。
- 导入前对全部条目进行前端校验。
- 导入成功后跳转到项目任务标签并重新查询。
- 导入失败时保留临时计划，不能清空用户编辑结果。
- ADMIN、普通 member 和非 owner 不显示入口。

### 5.7 任务列表

筛选：项目、负责人、状态、优先级、父任务、截止日期范围和标题搜索。

表格列：标题、项目、负责人、状态、优先级、进度、截止日期、更新时间和操作。

权限：

- ADMIN 只读。
- 项目成员可查看。
- owner 可创建、完整修改和删除符合条件的任务。
- owner 或 assignee 可修改任务状态。
- 已归档项目中的任务只读。

### 5.8 任务详情与日志

```text
任务标题                         [编辑] [修改状态] [删除]
项目 / 负责人 / 状态 / 优先级 / 截止日期 / 当前进度

任务描述

任务日志                                        [新增日志]
┌──────────────────────────────────────────────────────────┐
│ 操作人 │ 进度 │ 内容 │ 时间 │ 操作                         │
└──────────────────────────────────────────────────────────┘

[获取 AI 建议]
```

权限：

- 成员或 ADMIN 可查看。
- owner 可编辑和删除任务。
- owner 或 assignee 可修改状态和创建日志。
- owner 或日志创建者可删除日志。
- AI 建议仅 USER owner 或当前 assignee 可调用。
- ADMIN 不显示任何项目内容写入口。

AI 建议先显示为可编辑临时文本；用户人工修改并填写实际进度后，再通过普通任务日志接口提交。
AI 建议不能直接写入任务日志。

### 5.9 总结列表与详情

筛选：项目、任务、类型、创建者、分页和排序。

权限：

- 成员或 ADMIN 可查看。
- 项目成员可以创建总结。
- 创建者或 owner 可以修改和删除。
- ADMIN 只读。
- 已归档项目只读。

创建和编辑使用表单对话框；查看详情使用详情页面或抽屉。成功写入后重新查询列表和详情。

### 5.10 收到的邀请

仅 USER 访问。

列表内容：项目、邀请人、状态、邀请时间、响应时间和操作。

- PENDING 邀请可以接受或拒绝。
- 接受和拒绝均需确认。
- 成功后重新查询邀请列表。
- 接受成功后刷新项目列表。
- 已归档项目导致的失败不能改变当前邀请显示状态。
- ADMIN 访问时由路由守卫跳转 `/403`。

### 5.11 个人中心

所有已登录用户可访问。

内容：用户名、显示名称、手机、邮箱、系统角色、启用状态、修改资料和修改密码。用户名、系统
角色和启用状态只读。修改密码成功后所有 Session 失效，前端清空认证状态并跳转 `/login`。

### 5.12 403 和 404

403 说明当前账号无权访问页面，提供“返回工作台”和“返回上一页”，不显示后端安全细节。

404 说明页面不存在，提供“返回工作台”。登录后的 404 保留 AppLayout。

## 6. 公共反馈状态

所有列表和详情页至少实现以下状态。

### 6.1 Loading

- 路由认证初始化：全屏或根容器 loading，不能提前闪现登录页。
- 列表请求：表格区域 loading，保留筛选条件。
- 详情请求：详情骨架或内容区 loading。
- 提交请求：提交按钮 loading，禁止重复提交。

### 6.2 Success

- 查询成功：正常显示数据。
- 写入成功：使用一次轻量成功提示。
- 新增、修改、删除后重新查询或可靠同步页面数据。
- 不能只弹“成功”但保留旧数据。

### 6.3 Empty

- 无数据：显示空状态。
- 筛选结果为空：说明“没有符合条件的数据”，提供清除筛选。
- 初始无业务数据：提供当前用户有权限执行的主操作。
- ADMIN 只读页面无数据时不能显示项目内容创建入口。

### 6.4 Error

- 列表错误：错误说明和重新加载。
- 详情 404：资源不存在或不可见。
- 403：进入无权限页或显示明确无权限状态。
- 422：表单字段错误优先显示到字段。
- 409：保留页面数据，显示具体业务冲突。
- 未知网络错误：显示通用提示并允许重试。
- error 和 empty 不能同时出现。

## 7. 关键流程

### 7.1 未登录访问业务页

```text
访问业务 URL
→ Router 初始化认证状态
→ GET /api/users/me 返回 401
→ 跳转 /login?redirect=原业务地址
→ 登录成功
→ 回到原业务地址
```

只接受站内相对重定向地址，避免开放重定向。

### 7.2 已登录访问登录页

```text
访问 /login
→ 认证状态已经初始化
→ currentUser 存在
→ 跳转 /overview
```

### 7.3 USER 访问 ADMIN 页面

```text
访问 /admin/users
→ Router 检查 meta.roles
→ 当前 systemRole=USER
→ 跳转 /403
→ 不发送 GET /api/users
```

### 7.4 Session 失效

```text
业务请求返回 401
→ 统一清空认证状态
→ 跳转 /login
→ 保留安全的原页面 redirect
→ 页面不重复弹出多个错误消息
```

### 7.5 写操作成功

```text
打开表单或确认对话框
→ 前端校验
→ 获取 CSRF Header
→ 调用业务 API
→ 成功提示
→ 关闭对话框
→ 重新查询详情或列表
→ 页面显示服务端最终状态
```

## 8. 页面操作权限矩阵

| 操作 | ADMIN | USER owner | USER member | USER assignee |
| --- | --- | --- | --- | --- |
| 查看全部项目 | 是 | 仅参与项目 | 仅参与项目 | 仅参与项目 |
| 创建项目 | 指定 USER owner | 自动成为 owner | 自动成为 owner | 自动成为 owner |
| 编辑/归档项目 | 是 | 是 | 否 | 否 |
| 转移 owner | 是 | 是 | 否 | 否 |
| 直接添加成员 | 是 | 否 | 否 | 否 |
| 创建邀请 | 否 | 是 | 否 | 否 |
| 移除成员 | 是 | 是 | 否 | 否 |
| 创建/编辑/删除任务 | 否 | 是 | 否 | 否 |
| 修改任务状态 | 否 | 是 | 否 | 是 |
| 创建任务日志 | 否 | 是 | 否 | 是 |
| 创建总结 | 否 | 是 | 是 | 成员时可以 |
| 修改总结 | 否 | owner 或创建者 | 创建者 | 创建者 |
